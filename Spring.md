# JVAV SPRING PROJECT
## 기획
- 문제점을 찾는다
- 문제를 해결할 방법을 찾는다
- 사용자 입장에서 필요한 기능을 명세한다
- 기능을 구현하기 위한 정보를 생각해본다
### 시스템 아키텍쳐
- Multi-tier 아키텍쳐 -> 3티어 아키텍쳐(프리젠테이션, 비지니스, 데이터소스)
- 프리젠테이션 : 사용자와 소통(프론트)
- 비지니스 : 사용자와의 소통한것 처리(백)
- 데이터 소스 : 처리한것들이 저장되는곳(디비)

### 비지니스 아키텍쳐를 어떻게 만들것인가?
- Layered 아키텍쳐
1. UI 2. Application 3. Domain 4. Infrastructure
